# 后台菜单地图

这页按后台菜单解释每个页面是干什么的、什么时候会用、正常怎么配置、配完能拿到什么，以及需要交给开发者什么。

不同版本的后台菜单顺序可能会调整，但核心对象不变。

## 任务

**它解决什么问题**

任务页用于查看 Agent 每一次运行的结果和过程。上线后最常用的页面通常就是任务页，因为它回答“刚才发生了什么”。

**什么时候用**

- 用户反馈 Agent 回答不对。
- 业务系统说某次调用没收到结果。
- 工具调用失败。
- 想确认 Agent 是否看了知识库、是否调用了业务接口。
- 想把排障包发给开发者。

**你能看到什么**

- 会话列表。
- 每条任务的输入、路由、状态、结果摘要。
- 执行轨迹、工具调用、审批、知识引用、消息总账。
- debug bundle / debug report。

**配完能拿到什么**

任务页不是配置页，它产出的是排障信息：

- `job_id`
- `request_id`
- trace 事件
- debug bundle
- debug report

**给开发者什么**

当某次业务接入有问题，优先给开发者：

- `request_id` 或 `job_id`
- debug report
- 错误截图
- 对应路由 key

开发者再去看 [CONTRACT.md](../CONTRACT.md) 的 `/run`、`/jobs`、trace 和回调契约。

## 触发路由

**它解决什么问题**

触发路由定义一个 Agent 业务场景。每个业务能力都应该先落成一条路由，而不是让业务系统直接指定模型、工具和知识。

**什么时候用**

- 新增一个 Agent 场景。
- 修改某个场景使用的大脑。
- 给某个场景增加知识库。
- 控制某个场景能调用哪些业务工具。
- 调整会话连续性、预算、送达、重试和输入策略。

**正常怎么配置**

最小配置：

1. 填路由标识和名称。
2. 选择调度目标，初期通常用 `llm` 或 demo-agent。
3. 设置会话策略：
   - `new`：每次都是新任务。
   - `per_key`：按某个业务字段延续会话，如客户 ID、订单 ID、访客 ID。
   - `fixed`：固定会话。
4. 如需知识库，绑定知识库。
5. 如需调业务接口，绑定工具源并配置 allow scope。
6. 如需结果回传，配置 delivery 或 callback。

**配完能拿到什么**

- `route_key`：业务侧触发时要传的路由 key。
- 调用代码：后台会生成 curl/PHP/SDK 示例。
- 当前场景的工具白名单、知识库、输入策略和送达策略。

**给开发者什么**

- 路由的“调用代码”。
- 业务系统使用的接入方 token。
- 如果 route 使用 `per_key`，告诉开发者 metadata 里必须传哪个字段。
- 如果 route 使用工具源，告诉开发者 `subject_field` 对应哪个业务用户字段。

开发者继续看：

- [CONTRACT.md §1 POST /run](../CONTRACT.md#1-触发post-run)
- [QUICKSTART.md](../QUICKSTART.md)
- [TOOLS_DESIGN.md](../TOOLS_DESIGN.md)

## 接入方

**它解决什么问题**

接入方定义哪个业务系统可以调用中枢。它是业务系统的钥匙，不是最终用户身份。

**什么时候用**

- 一个新的业务系统要接入中枢。
- 要限制某个业务系统只能调用部分路由。
- 要给业务系统换 token。
- 要设置调用限速。
- 要停用某个业务系统接入。

**正常怎么配置**

1. 新建接入方。
2. 填 `app_id` 和名称。
3. 选择允许调用的路由。
4. 设置每分钟限速。
5. 保存后复制 token。

**配完能拿到什么**

- `app_id`
- 接入方 token
- 路由白名单

**给开发者什么**

- token：放在业务服务端环境变量，不要进前端。
- 允许调用的 route key。
- 调用示例。

开发者继续看 [CONTRACT.md §0 鉴权](../CONTRACT.md#0-鉴权三种凭证)。

## 调度目标

**它解决什么问题**

调度目标定义任务交给哪类大脑处理。

**什么时候用**

- 使用内置 `llm`。
- 注册一个远端执行器目标。
- 接入本地模型、内网 agent、OpenClaw、Codex、Claude Code、国内智能体工具、脚本或第三方运行时。

**正常怎么配置**

- `inhub`：中枢进程内目标。开源版内置 `llm` 和 demo-agent。
- `executor`：远端执行器目标。开发者在自己的机器上运行执行器认领任务；本地 OpenClaw、Codex、Claude Code、通义灵码/Trae 等国内智能体工具、自研 agent 或自动化脚本，都可以通过执行器协议接入中枢。

多数使用者不需要频繁进入这个页面。只要路由能选到目标即可。

**配完能拿到什么**

- target 名称。
- 目标类型。
- 是否需要项目目录。
- 超时配置。

**给开发者什么**

如果是 executor 目标，给开发者：

- target 名称。
- 执行器令牌。
- 中枢地址。
- 运行执行器的命令。

开发者继续看 [QUICKSTART.md 第 6 步](../QUICKSTART.md#旅程九步每步都有控制台页面) 和 [CONTRACT.md](../CONTRACT.md) 的 executor 相关契约。

## 模型凭证

**它解决什么问题**

模型凭证让中枢知道如何调用模型服务。它不是业务场景，只是模型能力的登记。

**什么时候用**

- 配云端大模型。
- 配 OpenAI-compatible 服务。
- 配本地 Ollama、vLLM、xinference 等兼容端点。
- 配视觉模型、语音模型、向量模型。
- 验证某个模型 key 是否可用。

**正常怎么配置**

1. 选择用途：对话、视觉、语音、向量等。
2. 填 base URL。
3. 填 API key。
4. 填默认模型。
5. 保存后点“验证”。

**配完能拿到什么**

- 凭证名称。
- 模型可达性验证结果。

**给开发者什么**

通常不需要把模型凭证给业务开发者。模型 key 应留在中枢或部署环境里。

如果开发者要配置路由，只告诉他凭证名称和模型用途即可。

## 工具源

**它解决什么问题**

工具源把业务系统的接口变成 Agent 可以治理调用的工具。

**什么时候用**

- Agent 需要查业务数据。
- Agent 需要创建业务对象。
- Agent 需要发起申请、修改状态、调用内部服务。
- 需要定义接口风险和审批策略。

**正常怎么配置**

1. 业务开发者提供 OpenAPI spec 或 SDK 生成的工具声明。
2. 在工具源里填 base URL、签名密钥和 spec。
3. 预览工具清单，确认哪些工具被识别。
4. 执行授权探针，确认业务侧不是只验签不授权。
5. 在触发路由里引用该工具源，并配置 allow scope。

**配完能拿到什么**

- 工具源名称。
- 工具源签名密钥。
- 工具清单。
- scope 列表。
- 授权探针结果。

**给开发者什么**

- 工具源签名密钥：业务系统验签用。
- 中枢调用业务接口时的签名规则。
- OpenAPI `x-agent-capability` 字段要求。
- 哪些 scope 会被哪些路由放行。

开发者继续看：

- [TOOLS_DESIGN.md](../TOOLS_DESIGN.md)
- [AI友好工具设计指南.md](../AI友好工具设计指南.md)
- [第三方对接指南.md](../第三方对接指南.md)
- [docs/examples/](../examples/)

## 知识库

**它解决什么问题**

知识库让 Agent 可以基于文档、制度、FAQ、产品说明回答问题。

**什么时候用**

- 需要回答制度、流程、产品说明。
- 需要基于文档内容做问答。
- 需要让网页聊天理解当前页面相关资料。

**正常怎么配置**

1. 先配置向量模型凭证。
2. 创建知识库。
3. 上传文档或配置数据源。
4. 等待切分和向量化。
5. 做命中测试。
6. 在触发路由里绑定知识库。

**配完能拿到什么**

- 知识库 ID。
- 可检索文档。
- 命中测试结果。

**给开发者什么**

通常不需要业务开发者改代码。只有当文档来自业务系统数据源时，需要开发者提供数据源接口或定时同步方式。

## 聊天入口

**它解决什么问题**

聊天入口把一条路由嵌入网页，让用户可以直接对话。

**什么时候用**

- 官网客服。
- H5 帮助助手。
- 业务后台里的 Agent 助手。
- 某个功能页上的页面上下文问答。

**正常怎么配置**

1. 新建聊天入口。
2. 绑定一条触发路由。
3. 设置标题、开场白、外观。
4. 设置 Origin 白名单。
5. 如需登录身份，配置票据签发方。
6. 如需上传附件，配置媒体存储和路由输入策略。
7. 复制嵌入代码。

**配完能拿到什么**

- `entry_key`
- `<script>` 嵌入代码
- 在线演示地址
- 票据签发说明
- 评价记录

**给开发者什么**

- 嵌入代码。
- 如果需要登录身份，给开发者票据签发规则。
- 如果页面需要上下文，告诉开发者是否要传 `page_key` 或让组件自动读取 URL。

开发者继续看 [CONTRACT.md §1.1 聊天入口](../CONTRACT.md#11-聊天入口网页同步问答的公开面)。

## 入站渠道

**它解决什么问题**

入站渠道让外部平台消息直接进入中枢，比如企业微信、钉钉、飞书等平台的机器人、应用消息或回调消息。

**什么时候用**

- 企业微信、钉钉、飞书等渠道消息要进中枢。
- 外部 IM 平台要接入中枢。
- 希望用户不用打开业务网页也能和 Agent 助手对话。

**正常怎么配置**

1. 新建渠道。
2. 选择渠道类型。
3. 填平台回调配置。
4. 绑定触发路由。
5. 到外部平台配置回调 URL。

**配完能拿到什么**

- 回调 URL。
- Token / EncodingAESKey 等渠道参数。
- route 绑定关系。

**给开发者什么**

如果是企业微信、钉钉、飞书等渠道，由配置人员在对应平台的管理后台填回调信息。开发者只需要看 [CHANNELS.md](../CHANNELS.md) 了解回调、回复窗口和送达限制。

## 媒体存储

**它解决什么问题**

媒体存储处理用户上传的图片、语音、文件。默认可用服务器本地存储；生产可换对象存储。

**什么时候用**

- 聊天入口需要上传图片。
- 用户会发语音。
- 用户会上传 PDF、Word、Excel 等文件。
- 需要 CDN、长期保存或多实例共享附件。

**正常怎么配置**

1. 默认可以先用本地存储。
2. 生产需要时登记对象存储桶。
3. 在聊天入口里关联存储桶。
4. 在触发路由里配置图片、语音、文件输入策略。

**配完能拿到什么**

- 存储桶名称。
- 附件 URL。
- 供路由多模态策略使用的素材地址。

**给开发者什么**

通常不需要业务开发者处理附件上传，除非业务系统希望使用自己的 CDN 或对象存储域名。

## 成本观测

**它解决什么问题**

成本观测帮助你看模型调用、工具调用、任务量和预算风险。

**什么时候用**

- 上线前评估成本。
- 某条路由调用量异常。
- 想限制某个场景每日 token 或费用。

**正常怎么配置**

- 在路由或接入方里配置预算闸。
- 在成本观测页查看使用情况。

**给开发者什么**

通常不给开发者敏感成本数据，只需要告诉他某个接入方或 route 的调用限制。

## 系统体检

**它解决什么问题**

系统体检检查配置结构、路由、工具源、执行器、自动分诊、死信队列和 Smoke。

**什么时候用**

- 新部署后。
- 导入演示数据后。
- 配完一批路由后。
- 上线前。
- 发现业务调用异常时。

**正常怎么用**

1. 先看错误和提醒。
2. 修复错误项。
3. 运行 Smoke。
4. 确认通过后再让业务接入。

**给开发者什么**

如果体检发现某个工具源、接入方、路由、执行器错误，把错误项截图或文本给对应开发者。

## 系统状态

**它解决什么问题**

系统状态用于看当前版本、数据库结构、契约版本和配置体检摘要。

**什么时候用**

- 部署升级后。
- 怀疑数据库结构没同步。
- 需要确认当前实例版本。
- 需要给开发者提供环境信息。

**给开发者什么**

- 应用版本。
- 数据库结构版本。
- 契约版本。
- 相关错误项。

## 后台账号

**它解决什么问题**

后台账号管理中枢控制台的使用者和权限。

**什么时候用**

- 新增管理员。
- 给只读人员开账号。
- 停用离职人员。
- 重置密码。

**注意**

后台账号只管中枢控制台，不等于业务系统的最终用户身份。

## 变更审计

**它解决什么问题**

变更审计记录后台配置被谁改过。

**什么时候用**

- 路由突然不可用。
- 工具源配置被改。
- 模型凭证变化。
- 需要追查某次配置变更。

**给开发者什么**

一般不直接给开发者，除非排查问题需要确认配置变更时间点。
