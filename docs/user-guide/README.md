# 使用者指南

这组文档面向准备在业务系统里落地 Agent 能力的使用者：业务负责人、产品经理、系统管理员、实施顾问和负责配置中枢的人。

它不替代开发者文档。开发者文档回答“接口怎么调、签名怎么验、SDK 怎么接”；使用者指南回答“我的业务想实现某个 Agent 能力，中枢里应该先配什么、为什么这么配、配完能交给开发者什么”。

## 适合谁读

- 你有一套已经在运行的业务系统，希望让 Agent 能查数据、答问题、处理部分业务动作。
- 你不想把 Agent 逻辑硬写进业务系统里，希望用一个独立中枢统一治理。
- 你需要和开发同事沟通：哪些接口要暴露、哪些动作要审批、哪些页面要嵌聊天组件。
- 你已经部署好百灵中枢，但不知道后台每个菜单到底解决什么问题。

## 立场说明

本文档只描述开源版百灵中枢的能力和使用方式。

开源版是一套单组织部署的中枢：一套后台、一个管理域、一套审计空间。它可以配置多个业务系统、多个接入方、多个路由、多个工具源和多个模型，但这些配置属于同一个中枢管理者。若多个组织之间需要完全隔离，建议分别部署多套开源版中枢。

## 读完后你应该能做什么

读完这组文档，你应该可以把一个业务需求拆成下面几类配置：

- 入口是什么：业务后端触发、网页聊天入口，还是企业微信、钉钉、飞书等入站渠道。
- 大脑是什么：云端模型、本地模型，还是远端执行器/自研 agent。
- Agent 能用哪些业务能力：一个或多个工具源里声明哪些接口，路由分别放行哪些 scope。
- Agent 回答时需要哪些资料：知识库、页面上下文、历史对话记忆。
- 结果怎么回到业务侧：轮询、callback、webhook、渠道送达。
- 高风险动作怎么处理：业务接口自身审批，还是中枢先冻结审批意图。

当这些内容在后台配置好后，你再把对应的调用代码、接入方 token、工具源签名密钥、聊天嵌入代码或回调验签说明交给开发者即可。
